# Week 2 – Encryption Algorithms

## Objective
Research and compare modern encryption algorithms to determine which is most suitable for the tool.

## Topics Covered
### Symmetric Encryption
- Uses a single shared key.
- Fast and efficient for large data.
- Examples: AES, DES, 3DES.

### Asymmetric Encryption
- Uses a public/private key pair.
- More secure but slower.
- Examples: RSA, ECC, Diffie‑Hellman.

## Algorithms Studied
- **AES** – Strong, efficient, widely used.
- **DES / 3DES** – Outdated and vulnerable.
- **RSA** – Secure but slow for large files.
- **ECC** – Strong security with smaller keys.
- **Diffie‑Hellman** – Used for key exchange.

## Challenges Identified
- Key management
- Brute‑force resistance
- Data integrity
- Performance
- Quantum threats

## Summary
AES‑256‑CBC was selected as the primary encryption method due to its strength, efficiency, and compatibility with OpenSSL.
